Update Instruction

To update your WDOS system, you will need two files: A compiled binary of the newer version of WDOS and the webpack compress file in .tar.gz format. You can get the two files on Github, private distribution servers or from cluster nodes. Please choose one of the update method below and press "Update" to start the update process.

Notes that updates only work if WDOS is started by the launcher. If not, you might need to manually update the system using system commands.

How to update manually?

If your system is not started by any launcher (versions before v1.120), you might want to manually update the WDOS following the steps below.

Linux (Debian)
Download the update of the latest WDOS using one of the above methods
SSH into your host, cd into the WDOS root (Usually located at ~/wdos/src/)
Check if the "updates" folder exists. If yes, check if the new WDOS binary and "system" and "web" folder exists
Backup any important files or config if needed
Stop the wdos service using sudo systemctl stop wdos
Copy the updates to the wdos root using cp -r ./updates/* ./
Restore the files or config
Start the wdos service using sydo systemctl start wdos
The new WDOS should be updated and ready.
Windows
Download the update of the latest WDOS using one of the above methods
Locate the cmd / powershell window that is running WDOS
Press Ctrl + C to gracefully stop WDOS
Open WDOS root and check if "updates" folder exists. If yes, check if the new WDOS binary and "system" and "web" folder exists
Backup any important files or config if needed
Copy all the file from the "updates" folder and overwrite the one in WDOS root (.\wdos\src\)
Restore the files or config
Start the new WDOS binary (exe) by double clicking it OR starting the start.bat file
